## Database Tables (MySQL)

1. `employees` ‚Äî employee_id, name, designation, base_salary, role (EMPLOYEE/MANAGER/ADMIN), last_promotion_date
2. `performance_reviews` ‚Äî review_id, employee_id, review_year, task_completion, attendance, team_collaboration, problem_solving, communication, leadership, client_satisfaction, total_kpi_score, category, created_at
3. `bonus_records` ‚Äî bonus_id, employee_id, review_year, total_kpi_score, category, bonus_percentage, bonus_amount, total_compensation, created_at
4. `users` ‚Äî user_id, employee_id (FK), username, password (BCrypt), role
5. `audit_logs` ‚Äî log_id, employee_id, action, performed_by, timestamp

## KPI System

7 KPI parameters with maximum points:

| KPI Parameter | Max Points |
|---------------|------------|
| Task Completion Rate | 25 |
| Attendance & Punctuality | 15 |
| Team Collaboration | 15 |
| Problem Solving | 15 |
| Communication | 10 |
| Leadership & Initiative | 10 |
| Client Satisfaction | 10 |
| **Total** | **100** |

**Formula:** `Total KPI = TC + AT + TCB + PS + COM + LD + CS`

### Bonus Tiers

| Tier | KPI Score Range | Bonus % |
|------|----------------|---------|
| Gold | 90 ‚Äì 100 | 20% |
| Silver | 75 ‚Äì 89 | 12% |
| Bronze | 60 ‚Äì 74 | 5% |
| No Tier | Below 60 | 0% |

### Calculations

- `Bonus Amount = Base Salary √ó Bonus% / 100`
- `Total Compensation = Base Salary + Bonus Amount`

## REST API Endpoints

| Method | Endpoint | Role | Description |
|--------|----------|------|-------------|
| POST | /api/auth/register | Public | Register user |
| POST | /api/auth/login | Public | Login, returns JWT |
| POST | /api/performances/calculate | MANAGER, ADMIN | Submit KPI & calculate bonus |
| GET | /api/bonuses/my | EMPLOYEE | View own bonus records |
| GET | /api/bonuses/all | ADMIN | View all employee bonuses |
| GET | /api/employees | ADMIN | List all employees |
| GET | /api/employees/{id} | ADMIN, MANAGER | Get employee details |

---

## RBAC (Spring Security + JWT)

- **EMPLOYEE** ‚Äî can only view own bonus records
- **MANAGER** ‚Äî can submit KPI reviews
- **ADMIN** ‚Äî full access: view all bonuses, all employees

---

## Validation Rules

- KPI scores cannot be negative or exceed their maximum
- Review year must be valid (not future)
- Employee must exist
- No duplicate review for same employee + same year

---

## OOP Requirements

To satisfy the 5-mark OOP section:

- Use an `enum` for `EmployeeCategory` (GOLD, SILVER, BRONZE, NO_TIER) and `Role` (EMPLOYEE, MANAGER, ADMIN)
- Use interfaces for repository layer (e.g. `EmployeeRepository`, `BonusRepository`)
- Use generics in service/utility classes where appropriate
- Use Java Streams for KPI score aggregation or filtering
- Custom exception classes: `EmployeeNotFoundException`, `DuplicateReviewException`, `InvalidKpiScoreException`
- Global exception handler using `@ControllerAdvice`
- Try-with-resources where applicable (e.g. file I/O or resource handling)
